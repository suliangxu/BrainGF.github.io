重定向

Posted on 2020-08-26 | In Linux - Linux基础

在Linux的任务中，我们常常会用到 nohup 命令来在后台运行某些服务，当在后台运行程序时，自然就会关注到程序的 输入、输出和错误 的问题。

一、文件描述符

Linux中一切皆文件，包括标准输入设备和标准输出设备在内的所有计算机硬件都是文件。

与输入输出有关的文件描述符主要有三个：0、1和2

文件描述符 文件名 类型
0 stdin 标准输入文件
1 stdout 标准输出文件
2 stdeer 标准错误输出文件

二、重定向

类型 符号 作用
标准输出重定向 command >file 覆盖，将正确输出结果输出到文件 file 中
标准输出重定向 command >>file 追加，将正确输出结果输出到文件 file 中
标准错误输出重定向 command 2>file 覆盖，将错误信息输出到文件 file 中
标准错误输出重定向 command 2>>file 追加，将错误信息输出到文件 file 中
正确输出和错误信息同时保存 command >file 2>&1 覆盖，把正确输出和错误信息同时输出到文件 file 中
正确输出和错误信息同时保存 command >>file 2>&1 追加，把正确输出和错误信息同时输出到文件 file 中
正确输出和错误信息同时保存 command >file1 2>file2 覆盖， 把正确输出结果输出到 file1，错误信息输出到 file2
正确输出和错误信息同时保存 command >>file1 2>>file2 追加， 把正确输出结果输出到 file1，错误信息输出到 file2
正确输出和错误信息同时保存 command >file 2>file 不推荐 （导致文件打开两次，引起资源竞争）
正确输出和错误信息同时保存 command >>file 2>>file 不推荐（同上
输入重定向 command <file 将文件 file 中的内容作为输入
输入重定向 command <<END 从标准输入（键盘）读取数据，直到分界符 END 停止（可自定义）
输入重定向 command <file1 >file2 将文件 file1 中的内容作为输入，并将处理结果输出到文件 file2

注：

  • 若不写文件描述符，在 < 后默认为0，在 > 后默认为1
  • 文件描述符与 < 、> 符号之间不允许有空格
  • 若既不想将输出结果保存到文件，又不想输出到屏幕，可以输出到 /dev/null 文件中
    • 可以将 /dev/null 看成系统垃圾箱

prefix

./configure命令执行完毕之后创建一个文件creating Makefile，供下面的make命令使用 执行make install之后就会把程序安装到我们指定的目录中去。

Configure是一个可执行脚本，它有很多选项，在待安装的源码路径下使用命令./configure –help输出详细的选项列表。其中 –prefix选项是配置安装的路径。

如果不配置该选项，安装后可执行文件默认放在/usr /local/bin，库文件默认放在/usr/local/lib，配置文件默认放在/usr/local/etc，其它的资源文件放在/usr /local/share，比较凌乱。

如果配置–prefix如：

可以把所有资源文件放在/usr/local/test的路径中，不会杂乱。用了—prefix选项的另一个好处是卸载软件或移植软件。当某个安装的软件不再需要时，只须简单的删除该安装目录，就可以把软件卸载得干干净净；移植软件只需拷贝整个目录到另外一个机器即可（相同的操作系统）。

当然要卸载程序，也可以在原来的make目录下用一次make uninstall，但前提是make文件指定过uninstall。
