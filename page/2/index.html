VC-bound：

坏事发生的几率很小。

对式子进行改写。

两个条件，使得Ein 接近 Eout：

1、好的的假设集（有break point）

2、够大的数据集（才能确保坏事发生的几率很低）

才能说能够学习。

3、加上一个好的算法，那么就可以说学到东西了。

4、再加上一点好的运气

VC维：给max non-break point 一个正式的定义。

如break point为k，dvc = k-1

如果不存在k，那么就说dvc无限大。

若N够大，且dvc>2，就可以认为增长函数小于N^dvc。

![image-20200216171529548](/Users/hhh/Library/Application Support/typora-user-images/image-20200216171529548.png)

与数据的分布无关，与目标长什么样也无关。

增长函数：

![image-20200215141454170](/Users/hhh/Library/Application Support/typora-user-images/image-20200215141454170.png)

假设：一个增长函数的最小的break point 是当 k = 2的时候：

break point = 2， 即说明没有一条线能够把某两个点完全分开。

当一个点时，有两种，两个点时最多有3种（因为break point为2），

所以三个点时，最多可以产生几种线呢？

1、只有一个二分的时候：

满足情况。没有线能够把某两个点完全分开。因为点的分布只有一种情况。

若想找到能将某两个点完全分开的线，最少需要4种二分。

![image-20200215144641407](/Users/hhh/Library/Application Support/typora-user-images/image-20200215144641407.png)

2、有4种二分的时候：

有可能存在：

![image-20200215144837400](/Users/hhh/Library/Application Support/typora-user-images/image-20200215144837400.png)

所以这4种划分，不能存在于假设集合当中。

所以找到4种二分时的另一种情形：

![image-20200215145028846](/Users/hhh/Library/Application Support/typora-user-images/image-20200215145028846.png)

3、但是当二分的数目大于等于5个时，会发现无论哪种划分，都会出现某两个点完全分开的情况，所以4种二分是极限。

![image-20200215145216699](/Users/hhh/Library/Application Support/typora-user-images/image-20200215145216699.png)

所以当存在break point时，似乎对于后来增长函数的趋势有一些限制，

能否找到一个多项式来表达假设集合里的最多的可能性，那么就可以认为这个感知器是可以学的到东西的。

上限函数：

已知一个增长函数在k的地方是break point，则请说出这个增长函数最多有多少种二分。

用向量表示。

如果可以找到这个上限函数，就可以不用考虑假说集中的细节问题。

比如：有两个增长函数，他们的break point都发生在3的地方，那么只要分析B(N, 3)的地方，就是上限函数在N个点的情况下，break point是3的情形。而不用对两个增长函数进行讨论。

![image-20200215150613492](/Users/hhh/Library/Application Support/typora-user-images/image-20200215150613492.png)

$\alpha + \beta$ 会小于等于 B（3，3），就是会小于有三个点的情况下，任意的三个点都能做二分。

05_Training versus Testing

Posted on 2020-02-14 | In 人工智能 - 机器学习基石

![image-20200214161650400](/Users/hhh/Library/Application Support/typora-user-images/image-20200214161650400.png)

假如假设：

训练的数据和测试假设（hypothesis）的数据，都来自同一个分布。

Ein()：在训练数据中假设的表现

Eout()：在测试集上假设的表现

二者来自于同分布。

若假设的集合不太大，并且数据量足够多的话，就能够确保对于每一个假设，Ein和Eout都会长得很像（近似相等），不论算法实际上是如何选择假设。

那么算法只要选择Ein最低的假设，若Ein很接近0的话，那么Eout也会很接近0，就达到了学习的效果。

![image-20200214162841322](/Users/hhh/Library/Application Support/typora-user-images/image-20200214162841322.png)

第一次课：

学习，就是又一个未知的f，我们希望通过算法能够找出一个g，使得g和未知的f非常的接近。

即 Eout(g) 要很接近0，就是对于测试集错误率越小越好。

第二次课：

在我们的测试集上做的很好就好了。就是可以找一些方法使得 Ein 很接近0.

第三次课：

我们其实在一个很特定的范围下面做学习。如批次的数据，监督式的学习，二元的分类等等

第四次课：

Ein 很接近 Eout：

如果假设的数量不是太多的话，有可能 Ein 和 Eout 可以很接近。

所以第二次课的事情，真的就可以使得 g和f很接近。

两个关键点：

1、到底 Ein 和 Eout 是不是会很接近

2、若二者可以有连接，那么如何使得 Ein 越来越小。

![image-20200214163705841](/Users/hhh/Library/Application Support/typora-user-images/image-20200214163705841.png)

M：假设集的大小（假设它是有限大的）

M较小时：上次证明，有不等式，M越小，坏事发生的概率就很小

M较大时：确实可能选到较好的假设，但是M较大，坏事发生的概率就增加了。

所以M其实是很重要的，应慎重选择。

所以无限多个M是不好的。

![image-20200214164056822](/Users/hhh/Library/Application Support/typora-user-images/image-20200214164056822.png)

假设可以使用有限大的 mH 来替换原来的 M。

所以从 mH 中学到一个较好的假设。

![image-20200214164232905](/Users/hhh/Library/Application Support/typora-user-images/image-20200214164232905.png)

使用 union bound 的方式，来计算坏事发生的概率。

但是，当 M 为很大的数字时，所以这个 union bound 就失效了。（加和的概率会很大）

union bound 的可行是假设坏事发生的概率不会太大，并且重叠的概率不太高。（可以通过切比雪夫不等式推到）

![image-20200214164825519](/Users/hhh/Library/Application Support/typora-user-images/image-20200214164825519.png)

因为不同的假设（若这两个假设都不太差的话）之间的误差是较为接近的，所以 他们发生坏事的部分就会比较接近，所以当他们进行 union bound 的时候，就会发生over-estimating （过分估计，过分大的上限），才会造成没有办法处理M无限大的情况。

所以我们必须找出这个（错误）重叠的部分，是否我们可以把这个无限多的M分成有限多个类别，每一类都差不多大，这样的话，我们也许能够找出哪些部分有重叠。

![image-20200214165625782](/Users/hhh/Library/Application Support/typora-user-images/image-20200214165625782.png)

所以如何将这些M个假设进行分类？

1、对于一个点来说，其实线只有两类：一类认为这个点为+1， 另一个认为是 -1

![image-20200214165908661](/Users/hhh/Library/Application Support/typora-user-images/image-20200214165908661.png)

2、对于两个点，那么则有四类：

![image-20200214170455766](/Users/hhh/Library/Application Support/typora-user-images/image-20200214170455766.png)

02_爬虫入门之爬虫基础了解

Posted on 2020-02-11 | In python全栈 - 爬虫

一、什么是爬虫

爬虫，即网络爬虫，大家可以理解为在网络上爬行的一只蜘蛛，互联网就比作一张大网，而爬虫便是在这张网上爬来爬去的蜘蛛，如果它遇到资源，那么他就会抓取下来。具体想抓取什么内容，这个由自己控制。

二、浏览网页的过程

在用户浏览网页的过程中，我们可能会看到很多好看的图片，比如【http://novel.tanzhuovip.com/】，我们会看到几张的图片以及百度搜索框，这个过程其实就是用户输入网址之后，经过DNS服务器，找到服务器主机，向服务器发送一个请求，服务器经过解析之后，发送给用户的浏览器HTML、JS、CSS等文件，浏览器解析出来，用户便可以看到形形色色的图片了。

因此，用户看到的网页实质是由HTML代码构成，爬虫便是这些内容，通过分析和过滤这些HTML代码，实现图片、文字等资源的获取。

三、URL的含义

URL，即统一资源定位符，也就是我们说的网址，统一资源定位符是对可以从互联网上得到的资源的位置和访问方法的一种简洁的表示，是互联网上标准资源的地址。互联网上的每个文件都有一个唯一的URL，它包含的信息指出文件的位置以及浏览器该怎么处理它。

URL的格式由三部分组成：

  • 第一部分是协议（或称为服务方式）

    http:// https:// ftp:// file://

  • 第二部分是存有该资源的主机IP地址（有时也包括端口号）

    127.0.0.1 www.tanzhouedu.com

    • 顶级域名和二级域名的区别：
      • 顶级域名：一级域名、根域名
        • 除了后缀（com、cn、com.cn、org.cn）以外只有一个点号
        • baidu.com baidu.com.cn shangfeng.club
      • 二级域名 www.baidu.com
      • 三级域名 a.www.baidu.com

  • 第三部分是主机资源的具体地址，如目录和文件名等。

爬虫爬取数据时必须要有一个目标的URL才可以获得数据，因此，它是爬虫获取数据的基本依据，准确理解它的含义对爬虫学习有很大帮助。

四、环境的配置

学习python，当然少不了环境的配置，该课程使用的版本为python2.7.9， 开发工具为pycharm

五、爬虫的重要思想

1、理论上来讲只要是网页上面能够看到的数据都是可以爬取的（因为所有看到的网页上的数据都是由服务器发送到我们电脑上的，只是有的数据加密过，很难解密）

2、在网页上无法看到或者无法获取的数据，爬虫同样不可能拿到，比如一些付费资料

3、分析网页数据的原则是从简到繁，从易到难

  • 直接通过网页源代码获取
  • 分析是否为ajax异步加载
  • 数据是否被加密

一、urllib与urllib2

python的urllib 和 urllib2 模块都做与请求URL相关的操作，但他们提供不同的功能。他们两个最显著的差异如下：

urllib2 可以接受一个Request对象，并以此可以来设置一个URL的headers，但是urllib只接受一个URL。这意味着，你不能伪装你的用户代理字符串等。

urllib模块可以提供进行urlencode的方法，该方法用于GET查询字符串的生成，urllib2的不具有这样的功能。这就是urllib 和 urllib2 经常在一起使用的原因。

二、常用的方法：

1、urllib2.urlopen()

urllib2.urlopen(url[, data][, timeout])

urlopen 方法是urllib2模块最常用也最简单的方法，它打开URL网址，url参数可以是一个字符串url或者是一个Request对象。URL没什么可说的，Request对象和data在request类中说明，定义都是一样的。

对于可选的参数timeout，阻塞操作以秒为单位。

urlopen方法也可以通过建立了一个Request对象来明确指明想要获取的url。调用urlopen函数对请求的url返回一个response对象。这个response对象类似于一个file对象，所以用.read()函数可以操作这个response对象，关于urlopen函数的返回值的使用，我们下面在详细说。

2、urllib2.Request()

urllib2.Request(url[, data][, headers],[, origin_req_host][, unverifiable] )

Request类是一个抽象的URL请求。

这里用到了urllib2.Request类，对于上例，我们只通过了URL实例化了Request类对象，其实Request类还有其他的参数。

data是作为参数的请求，如果data不等于None，则该请求为POST，否则为GET。

headers 是字典类型，头字典可以作为参数在request时直接传入，也可以把每个键和值作为参数调用add_header()方法来添加。作为辨别浏览器身份的User-Agent header是经常被用来恶搞和伪装的，因为一些HTTP服务只允许某些请求来自常见的浏览器而不是脚本，或是针对不同的浏览器返回不同的版本。例如：Mozilla Firefox浏览器被识别为：“Mozilla/5.0 （X11，U；Linux i686）Gecko/20071127 Firefox/2.0.0.11”。默认情况下，urllib2把自己识别为Python.urllib/x.y（这里的xy是python发行版的主要或次要的版本号，如在Python2.6中，urllib2的默认用户代理字符串是“Python.urllib/2.6”。下面的例子和上面的区别就是在请求时加了一个headers，模仿IE浏览器提交请求。

添加header也可以采用另一种方案：

req.add_header('Referer', 'http://www.python.org/')

4、HTTPCookieProcessor

很多网站的资源需要用户登录之后才能获取。

我们一旦登录后再访问其他被保护的资源的时候，就不再需要再次输入账号、密码。那么网站是怎么办到的呢？

一般来说，用户登录之后，服务器端会为该用户创建一个Session。Session相当于该用户的档案，该档案就代表着该用户。

那么某一次访问请求是否属于该用户呢？登录的时候服务器要求浏览器存储的一个Session ID的Cookie值。每一次访问都带上了该Cookie。服务器将Cookie中的Session ID与服务器中的Session ID比对就知道该请求来自哪个用户了。

5、opener

我们在调用urllib2.urlopen(url)的时候，其实urllib2在open函数内部创建了一个默认的opener对象。然后调用opener.open()函数。
